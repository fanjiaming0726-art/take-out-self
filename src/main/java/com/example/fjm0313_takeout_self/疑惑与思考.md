7. GET和POST，DELETE这些请求方式到底有什么区别，感觉如果前端提交的是DELETE，我在后端的DELETE接口里写get方法，那我实际上和GET所干的事情没有区别？
   答案：其实不同的请求方式的不同就在于提交参数的方式不同，GET是直接把参数放URL里面，其他的都是放body里面。
   这样的结果就是：
   （1）通过GET提交后，返回来的结果会被浏览器缓存，下次再访问同样的网址的时候就会直接把缓存中的结果返回给你，不会经过浏览器。而POST这些不会缓存，会直接打到服务器上
   （2）想要访问一个资源，把GET的URL直接复制再发过去就行，但是POST这些如果你复制URL之后它是不会起到和原来一样的效果的，因为body没了

20. 为什么每次用户请求结束的时候都需要显式销毁userId? 
因为ThreadLocal的工作流程是这样的，Tomcat在接收用户的请求的时候会分配线程池里面的一个线程，用户会在这个线程里面的ThreadLocal写入userId，如果不删除，下一个用户使用到的就是前一个用户的userId，数据就会串。
21. 为什么拦截器中的handler在第一步进行判断后不参与后续的操作？
因为（a instance of B b）是Java16后引入的一个语法糖，一旦发现a确实是B的对象，那么就会把a转型为b，并让b参与后续的操作
22. 我这个项目中有AOP的动态代理吗？
有的，当一个方法被@Transactional注解标记的时候，到时候调用这个方法的接口服务就是一个AOP代理对象，它负责来执行这个方法，并且在发生异常时回滚
23. 那我想问一下，是执行到这个方法的时候接口才会被代理对象注入，还是说在识别到这个@Transactional注解的时候就把接口注入进代理对象，往后所有方法都是这个代理对象在调用？
是后者，早在服务启动时就注入了代理对象，往后其他的方法的执行对象也是这个代理对象
24. AOP的原理是什么？
AOP的原理就是，被像@Transactional，@Before，@Around这样的注解标记完方法后，Spring在启动的时候就会检查这几个类，发现实现接口了就用JDK动态代理，发现没实现接口就用CGLIB代理，前者用接口实现来代理，后者用目标类子类作为代理，并被注入给这个类在别处的对象
AOP的概念是如何的呢：就是把所有方法的通用需求都交割一个人来管理，这个管理的人就是JDK代理对象

29. 实现了Redis的session共享后，Redis session在本项目中的具体作用是什么？
Redis Session会作为一个拦截器拦截一个动作，这个动作就是request.getSession.setAttribute("userId",userId)，这个动作原本会Tomcat内存里开辟出一块空间存储数据，但Redis会把这一步拦住，转而把数据存入redis中，之后的删除和获取操作全部通过Redis完成，确保多台服务器能够共享到用户session

32. 为什么Set<String> members = (Set<String>) (Set<?>)redisTemplate.opsForSet().members(BLACKLIST_KEY);这一步为什么不加(Set<?>)就是错的？
因为这是Java的泛型安全设计，你可以把Object强转成String，但是不能把Set<Object>转换为Set<String>，因为它不能让一个Integer数据放进去，所以需要一个桥梁，叫Set<?>，Java允许任何的Set转成Set<?>也允许Set<?>转成任何的Set。

34. 为什么Integer无法强转为Long类型？
