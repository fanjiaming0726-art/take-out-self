1. 为什么针对菜品的口味，份量需要单独再建一张表，一个新实体类，为什么不能直接一个dish表，一个dish实体类搞定所有问题
   答案：因为口味。份量这种字段属于是一对多的关系，如果在一列当中全部体现，看似比较方便，但不方便后续的提取，修改。所以针对新字段，可以新建一张菜品偏好表，用dish_id保持与原先菜品的联系，用id来区分到底是微辣还是中辣，是大份还是小份。如果没有dish_id的话，是不知道针对哪个菜品选口味的
2. HTTPServletRequest是根据前端传过来的字节流/json字符串映射到Tomcat服务器的吗？
   答案：是，HTTPServletRequest是Tomcat服务器根据前端传过来的请求信息字节流，将字节流信息打包成HTTPServletRequest的。我们需要用就在括号里面写参数，不需要就不写，但不管需不需要Tomcat服务器都会自动生成

   答案：不会，因为每一个用户获得的session是独立的

7. GET和POST，DELETE这些请求方式到底有什么区别，感觉如果前端提交的是DELETE，我在后端的DELETE接口里写get方法，那我实际上和GET所干的事情没有区别？
   答案：其实不同的请求方式的不同就在于提交参数的方式不同，GET是直接把参数放URL里面，其他的都是放body里面。
   这样的结果就是：
   （1）通过GET提交后，返回来的结果会被浏览器缓存，下次再访问同样的网址的时候就会直接把缓存中的结果返回给你，不会经过浏览器。而POST这些不会缓存，会直接打到服务器上
   （2）想要访问一个资源，把GET的URL直接复制再发过去就行，但是POST这些如果你复制URL之后它是不会起到和原来一样的效果的，因为body没了
8. 为什么update有update()，updateById()两种方法，而save却不需要划分
   答案：因为update有两种更新方式，根据wrapper匹配，根据实体的id匹配。而save不需要，只需要知道插谁就行了
   8.1 update()和updateById()有什么优劣？
   答案：update()可以一次对一个列表进行修改，而ById需要一个一个修改。更为便利点。
9. 为什么方法明确要求返回的是Result<AddrressBook>，可是我返回Result.file(String)类型的数据也可以，为什么不报错？
   答案：其实实际上是返回了Result<AddressBook>类型的，只是说object那一栏的值为null而已。如果是Result.success(String)，那么object的类型就会变为String，此时就会与AddressBook起类型冲突
10. 为什么给订单赋入地址的时候，不直接赋入一个addressBook，反而把字段设置成String类型并且给出一大段字符串拼接逻辑
    答案：因为addressBook和addressBookId相绑定，如果后期更改了id对应的地址，那么查询往期地址的时候就会查成更新后的那个地址，就搞错了
11. 为什么需要给地址字符串设置三元表达式
    答案：因为字符串为null时与其他字符串拼接会把null这个单词也拼进去
