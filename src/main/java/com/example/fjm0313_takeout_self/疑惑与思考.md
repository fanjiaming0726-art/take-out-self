   （2）想要访问一个资源，把GET的URL直接复制再发过去就行，但是POST这些如果你复制URL之后它是不会起到和原来一样的效果的，因为body没了

24. AOP的原理是什么？
    AOP的原理就是，被像@Transactional，@Before，@Around这样的注解标记完方法后，Spring在启动的时候就会检查这几个类，发现实现接口了就用JDK动态代理，发现没实现接口就用CGLIB代理，前者用接口实现来代理，后者用目标类子类作为代理，并被注入给这个类在别处的对象
    AOP的概念是如何的呢：就是把所有方法的通用需求都交割一个人来管理，这个管理的人就是JDK代理对象

（二）Redis
1. 为什么redisTemplate在更新缓存的时候都是用的delete方法而不是更新方法？
因为在并发状态下更新缓存可能会出现缓存一致性问题，Redis和MySQL不一样，Redis写数据的时候不加锁，导致请求写缓存的顺序可能和写数据库的顺序不一样，导致缓存数据被错误的数据覆盖掉。但delete方法不会出现这个问题。所以我选择在每一次查数据的时候没从数据库找见才写入缓存，而不是在数据更改的是时候写入缓存
2. 为什么滑动窗口需要使用ZSet，是为了使用它的根据时间范围删除旧缓存的功能吗？ZSet是什么结构，它是不是和Set一样是Redis的底层数据结构，目前我看数据结构都有在redisTemplate方法上有所体现，opsForValue使用的是什么数据结构？
    对，就是为了使用它的按照分数范围删除旧缓存的功能；ZSet是有序集合，根据score的分数进行排序；ZSet和Set、List、String、Hash结构并称Redis的底层常用数据结构，不那么核心的就是位图和Stream这种，只是听说过；opsForValue使用的就是String结构
3. 为什么滑动窗口比固定窗口好？
    因为设计出固定窗口就意味着你给了用户突破限流的机会，这个机会就在窗口即将过期的时候。所以导致那一瞬间的高并发，会对数据库的并发查询造成冲击
4. 现了Redis的session共享后，Redis session在本项目中的具体作用是什么？
    Redis Session会作为一个拦截器拦截一个动作，这个动作就是request.getSession.setAttribute("userId",userId)，这个动作原本会Tomcat内存里开辟出一块空间存储数据，但Redis会把这一步拦住，转而把数据存入redis中，之后的删除和获取操作全部通过Redis完成，确保多台服务器能够共享到用户session 
5. redisTemplate.opsForSet().add(BLACKLIST_KEY, userId.toString()); 为什么要toString()？
    是因为用的RedisTemplate是<String, Object>，反序列化的时候会把userId默认转型成Integer类型的数据，这样与之后校验的Long类型冲突 
6. 为什么RedisTemplate的opsForSet的这个结构删除的时候需要把key和值一块删了，而不是像delete那样只删key就行 
因为如果像delete那样删key会把set里面所有key全部删掉，必须得选key来删
7. 为什么Set<String> members = (Set<String>) (Set<?>)redisTemplate.opsForSet().members(BLACKLIST_KEY);这一步为什么不加(Set<?>)就是错的？
    因为这是Java的泛型安全设计，你可以把Object强转成String，但是不能把Set<Object>转换为Set<String>，因为它不能让一个Integer数据放进去，所以需要一个桥梁，叫Set<?>，Java允许任何的Set转成Set<?>也允许Set<?>转成任何的Set。

10. 为什么乐观锁不适合高并发？

（三）MySQL
1. 并发在这里的一个体现是数据库的并发查询，你能不能结合我的项目给我讲一下同一个查询接口在被多个用户同时使用时，数据库和Tomcat服务器会做什么动作？
    Tomcat会给每个用户一个线程，这三个线程同时访问Redis读取数据，读不到数据就去读数据库，数据库读取数据采用的是MVCC（多版本并发控制），所以可以同时读取数据不用加锁，但是如果需要同时写数据，例如执行update语句时，MySQL底层会加行锁，保证数据不被同时更改，执行完后再放下一个线程进来 
2. 行锁是什么，表锁是什么？
    行锁就是只锁表中的一行数据，通常在只修改一行数据的时候加；表锁就是锁整个表，MySQL在修改表的时候其他线程都不能进来 
3. MySQL是如何实现底层的事务的？结合我那个OrdersServiceImpl的事务方法聊一下？
    通过三个东西，undo log、redo log、锁
    undo log：在执行SQL语句前把数据原本的样子写进去，比如如果是插入语句，那么undo log里面就是id，如果是更新语句，那么就是这个数据原本的字对应原本的值
    redo log：把要做的修改写进去，再去修改内存中的数据页，这样能保证修改实际数据的时候MySQL断了，恢复后也能从redo日志继续操作
    锁：对于UPDATE这种需要上锁的SQL语句，锁的释放时机不在语句执行完，而在于整个方法COMMIT之后才释放锁
    如果中间有一步失败，就倒序回滚到先前的状态 
4. 那行锁是只要执行UPDATE语句都会加还是得识别出来多个SQL同时到达才加？
    只要执行UPDATE语句就会加锁，不管有没有并发 
5. 为什么会有并发查询压力，严重并发会给什么带来什么问题？数据库被打崩是什么意思？连接池被耗尽算问题吗，连接池和连接本身又没坏？
    因为同时有很多的请求进来等待处理。会给用户体验带来问题，被打崩就是数据库和服务器没有办法处理空余出来的请求了，导致用户的请求界面一直转圈最后超时报错。连接池被耗尽不会对连接池本身造成影响，但是没有办法处理新的请求了。
    设想这个场景就明白了，MySQL允许20个查询连接，tomcat允许100个请求连接，当20个请求连接匹配20个查询链接，而20个查询连接没有查完时，后面的请求就拿不到数据库连接，就会一直排队等着，导致服务器资源无法被释放给新的请求，所以新的请求一直进不来，页面一直转圈最后超时报错

（四）RabbitMQ
1. RabbitMQ 与 RocketMQ架构上有什么区别？
RabbitMQ : producer - exchange - queue - consumer
RocketMQ : producer - topic - consumer
2. consumer怎么知道producer消息是转发给谁的，这需要它们四个哪一个起作用？
producer指定一个exchange后，发一个queue的key给exchange，exchange去找哪一个queue有这个key，就把消息发给哪个queue，监听这个queue的consumer自然就知道消息了
3. exchange是不是有多个，每一个exchange都知道所有的queue吗？
的确有很多个，但是每个exchange都只认识自己绑定的queue，不认识其他exchange绑定的queue
4. 为什么明明写好了RabbitMQ的配置类，pom，application.yml。rabbitMQ的页面还是没有queue队列？
因为这是spring的懒加载机制，只有第一次使用到了queue的时候才会去创建queue。
5. 秒杀订单应该和正常的订单分离吧？
对，确实应该分离，已经新创建了一个seckillOrder实体类
6. RabbitMQ有几种交换机？
四种：按照匹配方式划分，有Direct/Headers/Fanout/TopicExchange四种，DirectExchange是精确匹配的那种
7. 为什么一个Bean方法里用了个参数，但是却说could not be autowired？
因为一个Bean方法里的参数，是需要在Spring容器里面的，如果这个参数是也是通过Bean获得的，那一定得是它自己而不是它的父类
8. 为什么会需要一个JacksonJson2MessageConverter呢？
因为这不同于SpringMVC返回给前端消息用HTTP，使用内置的Jackson序列化器进行Json转换，Rabbit MQ传输消息的时候走的是AMOP协议，如果不显示指定就会使用Spring AMOP来走Java原生序列化，因此得指定另一个
9. 我觉得JacksonJson2MessageConverter跟SpringMVC的内置的Jackson序列化器差不多，都是Jackson？
对，底层都是一个Jackson库
10. Jackson针对传输的消息是不是只有序列化和反序列化Json格式的作用？不同协议格式的消息流传的全过程是什么？
对的，Jackson只有Java对象 <-> Json字符串之间相互转换的作用，在不同的协议下，同样一份Json字符串，会被打包成不同的格式，根据不同的协议加上这样那样的请求头，最后一起被发到服务器上、
11. Jackson2JsonMessageConverter这个类，会转成字符串，并按照AMOP协议发过去？
不是，它只负责转成Json字符串，至于按照AMOP协议打包，就是RabbitMQTem【plate的事了
12. 削峰填谷是什么意思？这个项目里谁来削峰填谷？
削峰：将原本短时间内的高流量打低
填谷：在之后一段时间内按照合适数据库性能的速度，平稳的处理数据
Redis + Lua来削峰，RabbitMQ来填谷
