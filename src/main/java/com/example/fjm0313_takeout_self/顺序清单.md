---

## 从零编码顺序

### 第一阶段：项目骨架（与业务无关）

```
1. 创建 Spring Boot 项目，引入依赖（pom.xml）
2. application.yml（数据库连接、图片路径、MyBatis-Plus配置）
3. TakeoutApplication.java（启动类）
4. Result.java（common/，统一返回结果，所有接口都要用）
5. MybatisPlusConfig.java（config/，分页插件）
6. MyMetaObjectHandler.java（common/，自动填充createTime、updateTime）
7. WebMvcConfig.java（config/，先只写跨域配置和静态资源映射，拦截器后面再加）
```

**理由**：这些是基础设施，后面所有业务代码都依赖它们。

---

### 第二阶段：员工模块（商家/管理员）

```
建表 → employee表

8.  Employee.java（entity/）
9.  EmployeeMapper.java（mapper/）
10. EmployeeService.java（service/）
11. EmployeeServiceImpl.java（service/impl/）
12. EmployeeController.java（controller/，登录、登出、获取当前员工）
```

**理由**：先有管理员，才能登录后台去管理分类和菜品。

---

### 第三阶段：菜品分类模块

```
建表 → category表

13. Category.java（entity/）
14. CategoryMapper.java（mapper/）
15. CategoryService.java（service/）
16. CategoryServiceImpl.java（service/impl/）
17. CategoryController.java（controller/，增删改查）
```

**理由**：菜品属于某个分类，所以分类要在菜品之前。

---

### 第四阶段：菜品模块

```
建表 → dish表、dish_user_tips表

18. Dish.java（entity/）
19. DishUserTips.java（entity/，口味/份量）
20. DishMapper.java（mapper/）
21. DishUserTipsMapper.java（mapper/）
22. DishService.java（service/，定义saveWithFlavor等方法签名）
23. DishUserTipsService.java（service/）
24. DishUserTipsServiceImpl.java（service/impl/）
25. DishServiceImpl.java（service/impl/，实现saveWithFlavor、getByIdWithFlavor、updateWithFlavor）
26. DishDto.java（dto/，因为一个菜品对应多个口味，Dish装不下，需要DTO）
27. DishController.java（controller/，增删改查+按分类查）
28. CommonController.java（controller/，文件上传下载，菜品图片依赖它）
```

**理由**：菜品是核心业务数据，用户浏览和下单都依赖它。DishDto 在写 Controller 之前准备好，因为 Controller 的新增/修改接口接收的是 DishDto。

---

### 第五阶段：用户模块

```
建表 → user表

29. User.java（entity/）
30. UserMapper.java（mapper/）
31. UserService.java（service/）
32. UserServiceImpl.java（service/impl/）
33. UserController.java（controller/，注册、登录、登出、获取当前用户）
```

**理由**：有了用户，才能做购物车和下单。

---

### 第六阶段：登录拦截器

```
34. LoginCheckInterceptor.java（common/）
35. 回头改 WebMvcConfig.java（注册拦截器，配置放行路径）
```

**理由**：拦截器依赖 Session 中的 userId 和 employeeId，所以要等用户和员工的登录接口都写完，确认 Session 存取逻辑没问题之后再加拦截器，否则开发过程中每个接口都被拦下来，调试很痛苦。

---

### 第七阶段：地址模块

```
建表 → address_book表

36. AddressBook.java（entity/）
37. AddressBookMapper.java（mapper/）
38. AddressBookService.java（service/）
39. AddressBookServiceImpl.java（service/impl/）
40. AddressBookController.java（controller/，增删改查+设置默认地址）
```

**理由**：下单时需要选地址，所以地址在订单之前。

---

### 第八阶段：购物车模块

```
建表 → shopping_cart表

41. ShoppingCart.java（entity/）
42. ShoppingCartMapper.java（mapper/）
43. ShoppingCartService.java（service/）
44. ShoppingCartServiceImpl.java（service/impl/）
45. ShoppingCartController.java（controller/，加减菜品、查看、清空）
```

**理由**：下单时从购物车取数据，所以购物车在订单之前。

---

### 第九阶段：订单模块

```
建表 → orders表（含新字段）、order_detail表

46. Orders.java（entity/，含addressBookId临时字段）
47. OrderDetail.java（entity/）
48. OrdersMapper.java（mapper/）
49. OrderDetailMapper.java（mapper/）
50. OrdersService.java（service/）
51. OrderDetailService.java（service/）
52. OrdersServiceImpl.java（service/impl/）
53. OrderDetailServiceImpl.java（service/impl/）
54. OrdersDto.java（dto/，订单+明细列表）
55. OrdersController.java（controller/，用户下单、模拟支付、查看订单、商家接单/完成/取消）
```

**理由**：订单是整个系统的终点，它依赖用户、地址、购物车、菜品，所以放在最后。

---

### 总结：依赖链

```
项目骨架
  → 员工（管理后台的人）
    → 分类（管理员创建分类）
      → 菜品（管理员在分类下添加菜品）
        → 用户（来点餐的人）
          → 拦截器（保护接口）
            → 地址（用户填收货地址）
              → 购物车（用户选菜）
                → 订单（最终下单+支付+商家处理）
```

每写完一个阶段就启动项目用 Postman 测一遍，确保没问题再往下走。
