# 第一阶段：基础加固（Spring + MySQL）

1：HandlerInterceptor登录拦截器（√）
目前你每个 Controller 方法里都手动写 request.getSession().getAttribute("userId")，重复代码太多。用HandlerInterceptor 统一拦截，区分用户端和商家端权限。这是最该先做的。
优化前后：基于HTTPServletRequest的session手动重复获取userId -> 基于统一拦截器的自动获取userId

2：订单状态流转 + MySQL 事务（√）
目前订单只有下单，没有接单、配送、完成、取消的流程。加上状态机之后，涉及到订单状态变更 + 库存扣减 + 退款等多表操作，必须用 @Transactional 保证一致性。顺带深入理解 MySQL 的事务隔离级别、MVCC、redo log 和 undo log。

3：乐观锁防超卖（√）
给菜品加一个库存字段，下单时扣库存。多人同时下单同一道菜的时候用乐观锁（版本号机制）防止超卖。这里深入理解 MySQL 的锁机制：行锁、表锁、间隙锁、乐观锁 vs 悲观锁。

# 第二阶段：引入 Redis

4：Redis 缓存菜品和分类（√）
菜品列表和分类列表访问频率高但变动少，适合放 Redis 缓存。学习 Redis 的 String、Hash 数据结构，以及缓存穿透、缓存击穿、缓存雪崩的应对策略（包括互斥锁解决击穿问题）。
5：Redis 实现用户 Session 共享（√）
目前用的是 Tomcat 内置 Session，单机没问题，但如果以后要部署多台服务器就不行了。用 Spring Session + Redis 做分布式 Session。
6：Redis 黑名单 + 接口限流（固定窗口 -> 滑动窗口）（√）
用 Redis 的 Set 存黑名单用户，请求进来先查黑名单。用 Redis 的 INCR + EXPIRE 实现滑动窗口限流，未来或者结合 Sentinel 做更精细的接口限流、熔断和降级。
7：Redis ZSet 做销量排行榜（√）
用 ZSet 维护菜品销量排行，每次下单 ZINCRBY 加分，ZREVRANGE 取 Top N。这是 ZSet 最经典的应用场景。
8：Lua + Redis 实现秒杀
做一个限时特价菜品的秒杀功能。用 Lua 脚本在 Redis 中原子执行"判断库存 → 判断是否重复下单 → 扣库存 → 记录用户"，保证高并发下不超卖。这里深入理解 Redis 的单线程模型和 Lua 脚本的原子性。

# 第三阶段：引入消息队列

9：RabbitMQ 异步下单
秒杀场景中，Lua 脚本只做资格校验和预扣库存，真正的订单入库操作通过 RabbitMQ 异步处理。用户秒杀成功后立即返回"排队中"，消费者慢慢消费消息写入数据库。这就是削峰填谷。
10：RabbitMQ 延迟队列做订单超时取消
用户下单后 15 分钟未支付自动取消。用 RabbitMQ 的死信队列或延迟插件实现定时取消，恢复库存。
11：RabbitMQ 商家端消息推送
用户下单后通过 MQ 通知商家端有新订单，商家端通过 WebSocket 实时接收。

# 第四阶段：搜索与存储扩展

12：Elasticsearch 菜品搜索
菜品多了之后 MySQL 的 LIKE 查询太慢。引入 ES 做全文检索，支持按菜名、描述、分类模糊搜索，支持拼音搜索和搜索建议。数据从 MySQL 同步到 ES 可以用 Canal 或者 MQ。
13：MongoDB 存储用户评价
订单完成后用户可以评价，评价内容是非结构化的（文字 + 图片 + 评分 + 标签），用 MongoDB 存储比 MySQL 更灵活。同时学习 MongoDB 的文档模型和聚合查询。

# 第五阶段：微服务与网关
14：拆分微服务 + Gateway 网关
项目大了之后拆成用户服务、菜品服务、订单服务、支付服务。用 Spring Cloud Gateway 做统一入口，路由分发到各个服务。在网关层做统一鉴权、限流、日志。
15：Gateway 实现文件断点续传
菜品图片上传如果文件大，需要断点续传。在网关层处理分片上传和续传逻辑。
16：Sentinel 熔断降级
下游服务挂了不能拖垮整个系统。比如支付服务超时了，订单服务要快速降级返回"支付繁忙请稍后"，而不是一直等着。

# 第六阶段：AI + 移动端

17：AI Agent 智能点餐
接入大模型做智能点餐助手，用户可以说"推荐一个不辣的菜"或者"帮我凑一个 50 元以内的套餐"，AI 根据菜品数据和用户历史订单做推荐。需要用到 Function Calling，让 AI 调用你的菜品查询和下单接口。
18：打包成安卓 APP
前端用 Vue 或 React 写好页面后，用 Capacitor 或者 uni-app 打包成安卓 APK。后端接口不需要改，APP 直接请求你的后端 API。
