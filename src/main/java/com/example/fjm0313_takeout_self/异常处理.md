（一）异常：
1. 测试接口时返回的是[]而不是json字符串
排查：发现是Result类未加@Data注解，导致序列化为json的时候，无法调用Result的getter方法赋值
解决：使用@Data注解
2. NoSuchBeanDefinition异常
排查：这是bean对象未注册进容器的异常，通过报错信息得知是“com.example.fjm0313_takeout_self.mapper.EmployeeMapper”没有被注册进容器，检查发现是因为没有加@Mapper注解
解决：使用@Mapper注解
3. 明明登录成功，测试current接口的时候，却返回未登录提示信息
排查：这是由于setAttribute时，使用了user.getId(),而user只含有name和password两种信息，导致id为空，通过空id查出了null
解决：更换为实际数据库的用户id
4. cannot be inherited with different type arguments:
排查：这是由于ServiceImpl<>的第二个参数和后面实现的接口的参数类型不一致
解决：对齐
5. Cause: java.sql.SQLIntegrityConstraintViolationException: Duplicate entry '��������' for key 'dish.name'
; Duplicate entry '��������' for key 'dish.name'] with root cause
解析：重复插入字段
排查：插入的菜名已存在 
6. version = version + 1WHERE id = ?
解析：sql语法不对
排查：dishMapper的version + 1和WHERE之间没空格、
解决：加空格
7. 'dependencies.dependency.version' for org.springframework.boot:spring-boot-starter-webmvc:jar is missing. @ line 36, column 21
解析：springboot里面的parent模块的版本丢失
解决：优先下载parent模块
8. 在依据Redis版本下降spring boot版本后，突然间所有文件都报了java file is located outside of the module source root,so it won't be compiled错误
解析：其实不是因为文件位置的问题，而是因为springboot降级后，原先的mybatis-plus和spring boot的某些依赖标签不可用，因为这些标签在就版本中不可见，所以导致依赖全部失效
9. [ERROR] Failed to execute goal org.apache.maven.plugins:maven-compiler-plugin:3.11.0:compile (default-compile) on project Fjm0313_takeout_self: Resolution of annotationProcessorPath dependencies failed: version can neither be null, empty nor blank -> [Help 1]
解析：没法处理version，原因是build里面没有显式指定版本
解决：手动指定版本号version
10. /F:/IdeaProjects/Fjm0313_takeout_self/src/main/java/com/example/fjm0313_takeout_self/config/LoginInterceptor.java:[11,32] 程序包org.jspecify.annotations不存在
解析：旧版本里没有相应的注解包

12. Remote host terminated the handshake
解析：阿里云镜像的SSL连接不稳定，应该是因为开了VPN的缘故
解决：关掉VPN重新下载
13. failed to transfer during a previous attempt ，This failure was cached in the local repository and resolution is not reattempted until the update interval of mirror has elapsed or updates are forced
解析：上一次数据传输就失败了，失败的结果被缓存了，在下一次强制更新前不会重新尝试下载
解决：删除缓存文件

17. Caused by: java.lang.SecurityException: Attempt to deserialize unauthorized class com.example.fjm0313_takeout_self.common.MQ.SeckillMessage; add allowed class name patterns to the message converter or, if you trust the message originator, set environment variable 'SPRING_AMQP_DESERIALIZATION_TRUST_ALL' or system property 'spring.amqp.deserialization.trust.all' to true
解析：这是RabbitMQ的反序列化安全机制，spring AMOP默认不允许反序列化任意Java对象
解决：往RabbitMQConfig类中加入一段代码：
@Bean
public MessageConverter messageConverter() {
return new Jackson2JsonMessageConverter();
}
